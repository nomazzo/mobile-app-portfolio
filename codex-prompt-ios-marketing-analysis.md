# Codex Prompt - iOS App Marketing Feature Analysis

## 使い方

1. VS Codeで解析対象のiOSアプリのリポジトリ/フォルダを開く。
2. このプロンプトをCodexに渡す。
3. `ios-marketing-feature-analysis-template.md` を `docs/ios-marketing-feature-analysis.md` として埋めてもらう。
4. 生成された文書をChatGPTに渡し、App Store用のApp名、Subtitle、Promotional Text、Description、Keywords、What's New、スクリーンショットコピーを作成する。

---

## Codexへの依頼文

```text
あなたは、iOSアプリのソースコードを読み取り、App Storeメタデータ作成に使うための「マーケティング向け機能分析ドキュメント」を作成するアシスタントです。

目的:
- このアプリのユーザー向け機能、価値、利用シーン、訴求ポイント、スクリーンショットで見せるべき内容、キーワード素材を整理する。
- 後でChatGPTにこの文書を渡し、App Storeのアプリ名、サブタイトル、プロモーションテキスト、説明文、キーワード、What's New、スクリーンショットコピーを作成する。

重要:
- Android移植仕様書ではありません。
- 実装詳細の網羅ではなく、App Store掲載文言を作るためのマーケティング寄り情報を整理してください。
- ただし、推測だけで書かず、確認できたファイル・画面・関数・リソースを根拠として残してください。

作成先:
- `docs/ios-marketing-feature-analysis.md`

使用テンプレート:
- `docs/ios-marketing-feature-analysis-template.md` が存在する場合はそれを使用してください。
- 存在しない場合は、以下の構成で新規作成してください。

解析対象:
- 現在VS Codeで開いているiOSアプリのフォルダ全体。
- Storyboard / XIB / SwiftUI / Objective-C / Swift / Info.plist / Localizable.strings / Assets / README / 設定ファイルを確認してください。

必ず確認してほしいもの:
1. アプリ名、Bundle ID、カテゴリ推定
2. 主要画面、主要ViewController/SwiftUI View
3. ユーザーが実際に使う機能
4. 写真/カメラ/編集/保存/共有/設定/課金/広告などの有無
5. App Store説明文に使える強い訴求ポイント
6. スクリーンショットで見せるべき画面・流れ
7. キーワード候補の素材語
8. ローカライズ文言から読み取れる機能
9. IAPや広告など、メタデータでの扱いに注意すべき要素
10. 不明点・手動確認が必要な点

作業ルール:
- ソースコードは変更しないでください。
- 既存iOSコードをリファクタリングしないでください。
- ビルドは行わなくて構いません。最終確認はXcodeで人間が行います。
- 読み取り専用の調査コマンドは使ってよいです。
- ファイル削除、git commit、git push、git reset、git clean、外部ネットワークアクセス、依存関係追加は行わないでください。
- 使われていない古いコード、コメントアウトされた機能、未完成の機能は、現行機能として断定しないでください。
- 画面やコードから確実に確認できない機能は「未確認」または「可能性」として記載してください。
- App Store向け文言として誇大表現になる表現は避け、後でChatGPTが安全にコピーを作れるように素材として整理してください。

推奨する調査手順:
0. 旧アプリ概要を確認する（末尾に掲載）
1. ルートフォルダの構成を確認する。
2. Xcode project / workspace / Info.plist / Storyboard / XIB / SwiftUI entry pointを確認する。
3. 主要ViewController、View、Manager、Service、Helperを確認する。
4. Localizable.strings、Assets、画像名、ボタン名、タブ名を確認する。
5. IAP、広告、レビュー依頼、保存/共有、カメラ/写真権限などのユーザー向け機能を確認する。
6. ユーザー価値、訴求軸、スクショ案、キーワード素材に整理する。
7. `docs/ios-marketing-feature-analysis.md` を作成する。

出力形式:
- Markdown
- 日本語で記載
- App Storeコピー作成に使いやすいように、ユーザー向け表現を中心にする
- ただし根拠ファイル/画面/関数はコード上の名称で明記する

ドキュメントに必ず含めるセクション:

# <App Name> - iOS Marketing Feature Analysis

## 0. 解析メタ情報
- 現在のアプリ名
- Bundle ID
- 想定カテゴリ
- 主な実装言語
- 主要画面定義
- 解析対象の主なパス
- 解析できなかった範囲

## 1. アプリの一言要約
- ユーザー向け一言説明
- アプリの本質
- 何のアプリではないか

## 2. 想定ユーザーと利用シーン
- 想定ユーザー
- 主な利用シーン
- 使用頻度の想定

## 3. ユーザー向け主要機能
- 機能名
- ユーザーにとっての価値
- 根拠ファイル/画面/関数
- 訴求に使えるか

## 4. 画面・ユーザーフロー分析
- 主要画面
- 基本フロー
- スクリーンショットで見せるべき流れ

## 5. 差別化ポイント・強み
- 競合と比べたときの訴求軸候補
- アプリの雰囲気・ブランドトーン

## 6. App Store メタデータ作成のための素材
- App名候補の方向性
- サブタイトル向け訴求軸
- プロモーションテキスト向け訴求軸
- Descriptionに入れるべき内容
- Keywords作成用の素材語

## 7. スクリーンショットコピー作成のための素材
- 1枚目の最有力訴求
- 5枚構成案
- Before/Afterや作例が必要か

## 8. 課金・広告・プレミアム機能の扱い
- 広告の有無
- IAPの有無
- メタデータで避けるべきこと

## 9. ローカライズ時の注意
- 翻訳しないほうがよい語
- 言語によって表現を変えるべき内容
- 英語マスターコピーの方向性

## 10. 証拠・根拠メモ
- 判断と根拠ファイル/画面/関数

## 11. 未確認・手動確認が必要なこと

## 12. ChatGPTへ渡すときの要約

最後に、以下の報告フォーマットで作業結果を返してください。

- ...

作成/更新ファイル:
- `docs/ios-marketing-feature-analysis.md`

主な確認ファイル:
- ...

未確認/注意:
- ...
```
