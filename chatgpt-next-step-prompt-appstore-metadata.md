# ChatGPT Prompt - App Store Metadata from Marketing Feature Analysis

## 使い方

Codexが作成した `docs/ios-marketing-feature-analysis.md` をChatGPTに貼り付けたあと、以下のプロンプトを使う。

---

## ChatGPTへの依頼文

```text
以下は、iOSアプリのソースコードをCodexで解析して作成したマーケティング向け機能分析ドキュメントです。
この情報をもとに、App Store更新用のメタデータ案を作成してください。

目的:
- 既存iOSアプリの更新用に、App Storeで使う文言を作る。
- 誇大表現を避け、実装済み機能に基づいた自然な訴求にする。
- 古いアプリの印象を改善し、検索流入とCVR改善を狙う。

作成してほしいもの:
1. App名候補 10案
   - 30文字以内を意識
   - 検索語を入れる案とブランド寄り案を混ぜる
2. Subtitle候補 10案
   - 30文字以内
   - 機能・ベネフィットが伝わるもの
3. Promotional Text候補 5案
   - 170文字以内
4. Description 英語版または日本語版の草案
   - 冒頭で価値を伝える
   - 主要機能を箇条書きで説明
   - 最後に自然な利用促進文
5. Keywords候補
   - 100文字以内の最終候補を3パターン
   - 重複語を避ける
   - 関連性が低い語を入れない
6. What's New候補 3案
   - 既存アプリ更新向け
   - SDK更新、安定性向上、広告/プライバシー対応、UI改善などを自然に表現
7. スクリーンショット用キャッチコピー 5枚分
   - 1枚目は最重要訴求
   - 短く、AppScreensに入れやすい文言

注意:
- ドキュメントに根拠がない機能は強く訴求しない。
- 「最高」「No.1」「完全無料」など根拠が必要な表現は避ける。
- 課金や広告については、ユーザー価値がある場合のみ自然に触れる。
- App名やSubtitleは、自然さとASOのバランスを取る。

出力形式:
- 見出しごとに整理
- そのままコピーしやすい形
- 可能なら最後に「推奨セット」を1つ選んで理由を説明
```
